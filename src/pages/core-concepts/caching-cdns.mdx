# Caching and CDNs

## Caching

Caching involves storing copies of frequently accessed data in a temporary, fast-access storage layer (usually RAM) to reduce latency and database load.

**Cache Eviction Policies:**
- **LRU (Least Recently Used):** Discards the least recently accessed items first.
- **LFU (Least Frequently Used):** Discards the items accessed least often.
- **FIFO (First In First Out):** Discards the oldest items first.

**Write Policies:**
- **Write-Through:** Data is written to the cache and the database simultaneously. (Safe, slower writes).
- **Write-Back (Write-Behind):** Data is written to the cache alone, and asynchronously synced to the database later. (Fast writes, risk of data loss if cache crashes).

## CDN (Content Delivery Network)

A CDN is a geographically distributed network of proxy servers and their data centers. The goal is to provide high availability and performance by distributing the service spatially relative to end-users.

CDNs are primarily used for static assets:
- Images, Videos
- CSS, JavaScript files
- HTML pages

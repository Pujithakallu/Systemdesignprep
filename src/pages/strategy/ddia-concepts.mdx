# DDIA Core Concepts

*Designing Data-Intensive Applications* (DDIA) by Martin Kleppmann is considered the bible of system design. Here are the core concepts you must know for senior-level interviews.

## 1. Reliability, Scalability, and Maintainability
- **Reliability:** The system continues to work correctly even when things go wrong (hardware faults, software bugs, human error).
- **Scalability:** The system can cope with increased load (data volume, traffic volume, complexity) without performance degradation.
- **Maintainability:** The system is easy for new engineers to understand, operate, and modify.

## 2. Replication Strategies
How do we keep multiple copies of data in sync?
- **Single-Leader (Active/Passive):** One node takes all writes, followers copy the data. Easy to reason about, but the leader is a bottleneck.
- **Multi-Leader (Active/Active):** Multiple nodes accept writes. Great for multi-datacenter setups, but requires complex conflict resolution.
- **Leaderless (Dynamo-style):** Any node accepts writes. Uses "Quorum" reads/writes to ensure data consistency (e.g., Cassandra).

## 3. Partitioning (Sharding)
How do we split data across multiple machines?
- **Key Range Partitioning:** Keys A-M on Node 1, N-Z on Node 2. Good for range queries, bad for hotspots.
- **Hash Partitioning:** Hash the key to determine the node. Great for even distribution, bad for range queries.

## 4. Transactions and Isolation Levels
How do we handle concurrent access to data?
- **Read Uncommitted:** Dirty reads allowed.
- **Read Committed:** No dirty reads.
- **Repeatable Read (Snapshot Isolation):** If you read a row twice in a transaction, it will be the same. Usually implemented via MVCC (Multi-Version Concurrency Control).
- **Serializable:** The strongest level. Transactions act as if they were executed sequentially, one after another.

## 5. Consensus Algorithms
How do distributed nodes agree on something (like who the Leader is)?
- Algorithms like **Paxos** and **Raft** ensure that a cluster of nodes can agree on a single state, even if some nodes fail or network packets are delayed.
